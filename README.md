# 欠料表回复工具 v1.0

基于 **C# (.NET 8) + WPF + ClosedXML** 开发的桌面软件，用于把 ERP 导出的欠料表，按
**外协 / 外购** 两个板块自动生成可直接下发的「欠料回复单」。

## 功能

- 解析欠料表（.xlsx，33 列，近万行秒级读取）
- **外购（W列=N）**：按 `采购员`（U列）拆分；多人合并（如 `李婧婧,龚慧伦`）只取第一个，空值归「未分配」
- **外协（W列=Y）**：按 `供应商名称` 拆分。解析优先级：① T列(供应商未清PO明细)第2段；② T列为空时取 Z列(SRM未生效PO)第2段；③ 都无则归「待指定供应商」（尚未下单的外协欠料，需人工分配）
- 每个对象生成一张欠料回复单（物料清单 + 交期 + 结余 + 合计）
- 支持 **分文件导出**（每个对象独立 xlsx，按板块分子目录）或 **单文件多 sheet**
- 勾选式选择需要导出的分组

## 构建与运行

> 本机需安装 [.NET 8 SDK](https://dotnet.microsoft.com/download/dotnet/8.0)（Windows x64）。

```powershell
# 在项目目录（含 .csproj 的目录）执行：
dotnet restore
dotnet build -c Release
# 运行
dotnet run -c Release
```

### 发布为单文件 exe（推荐给采购同事使用）

```powershell
dotnet publish -c Release -r win-x64 --self-contained false -p:PublishSingleFile=true -o ./publish
```

`publish/` 目录下生成的 `欠料表回复工具.exe` 即为独立可执行文件
（需目标机器已装 .NET 8 运行时；若要完全独立，把 `--self-contained false` 改为 `true`）。

## 使用步骤

1. 打开软件，点「选择欠料表并解析」载入 `20260717欠料表.xlsx`
2. 列表中显示所有分组（外协供应商 / 外购采购员）及明细条数
3. 勾选要导出的分组，点「导出回复单」选择文件夹
4. 在导出文件夹得到 `外购/`、`外协/` 子目录（或单文件），每个对象一份回复单

## 输出列说明（回复单）

物料 / 名称 / 型号 / 规格 / 品牌 / 需求汇总 / 发货需求结余 / 生产+发货结余 /
生产+发货+备料结余 / 供应商回复交货日期 / 供应商未清PO明细 / 采购员(下单人) /
物料组描述 / 领料工序，并附合计行。

## 外购采购员筛选用法

导入欠料表后，在「导出设置 → 外购采购员筛选」框中输入采购员名字：
- 留空 = 导出全部外购分组；
- 输入单个名字（如 `龚慧伦`）只导出该采购员；
- 多个用逗号分隔（如 `李婧婧,王伟`）；
- 支持模糊匹配（包含即可，如输 `龚` 会匹配所有含「龚」的采购员）；
- 也可直接点右侧下拉框从已解析的采购员名单中选择（可编辑）。

> 注意：筛选仅在**外购板块**生效；外协仍按供应商全量导出（可勾选具体供应商）。

## 已知业务问题（待你确认）

1. **供应商名称不统一**：ERP 中同一供应商在 T列(全称，可能截断如「武汉信瑞达配…」)与 Z列(简称如「信瑞达」)写法不同，会被拆成两份回复单。如需合并，请提供「供应商别名映射表」，我可加入归一化。
2. **外协「待指定供应商」1824 行**：这些外协欠料既无 T列也无 Z列 PO 信息（尚未下单），无法自动判定供应商，统一归入「待指定供应商」回复单，供你人工分配。
3. **外购「未分配」395 行**：U列采购员为空的行归此组。
